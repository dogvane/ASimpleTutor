## 1. 界面布局草稿 (Wireframe)

## 2. 交互逻辑详解

### 2.1 幻灯片浏览 (Slide Navigation)

*   **Slide 定义**：右侧不仅是一张静态图片，而是一个 HTML 容器（Card）。
*   **串联模式**：当用户在左侧点击“2.1 基础概念”这一父节点时，右侧会自动生成一个播放列表（Playlist），包含该章节下所有子知识点的 Slide。
*   **翻页操作**：
    *   **鼠标滚轮**：自然滚动。向下滚动到底部后，继续滚动则自动切换到下一张 Slide（类似抖音网页版或 PPT 浏览体验）。
    *   **键盘操作**：支持 `Space`, `Enter`, `↓`, `→` 切换下一页。
    *   **点击区域**：卡片右边缘点击 -> 下一页，左边缘点击 -> 上一页。

### 2.2 原文悬停 (Source Hover)

*   **触发**：在 Slide 的正文文本中，部分文字下方会有虚线或高亮标记。
*   **反馈**：鼠标悬停（Hover）在这些区域时，光标旁会浮现一个轻量级的 Tooltip 或 Floating Card。
*   **内容**：显示扫描到的原始 Markdown/文档片段，证明该知识点的来源。移开鼠标即消失，不打断阅读流。

### 2.3 知识点跳转 (Knowledge Hyperlink)

*   **场景**：当讲解“神经元”提到了“激活函数”，而“激活函数”是另一个独立的 Knowledge Point (KP) 时。
*   **交互**：文字显示为链接样式（如蓝色下划线）。
*   **动作**：点击链接 -> 右侧 Slide 视图直接**跳转**到“激活函数”对应的 Slide 页面。
    *   *历史记录*：系统保留跳转前的浏览位置，可以通过浏览器“后退”或界面上的“返回”按钮回到之前的知识点。

### 2.4 嵌入式交互 (Embedded Interaction)

Slide 内部不仅仅是文本，还可以包含：
*   **代码块**：支持一键复制，如果接通了执行环境，支持简单的 `Run`。
*   **图表**：可缩放查看细节。
*   **小测验**：通过 `next slide` 之前的转场页，插入一个“随堂一问”，用户点击选项后直接在卡片上显示 ✅ 或 ❌ 及解析。

### 2.5 音频播报 (Audio Dictation) [预留功能]

*   **功能**：为每一页 Slide 提供对应的语音讲解。
*   **交互**：
    *   **播放控制**：界面右上角提供小喇叭图标 [🔈]，仅当系统检测到当前页存在对应的本地音频文件时点亮/可用。
    *   **内容来源**：优先查找本地知识点资源目录下的预生成 `.mp3` 文件。
    *   **机制说明**：音频 **不需要** 在浏览时实时生成（TTS），而是在知识点内容的后台生成阶段一并生成并保存。
    *   **同步**：翻页时，如果开启了自动播放，系统会自动检索是否有对应的 `.mp3`，有则播放，无则静默。

## 3. 视觉风格建议

*   **极简主义 (Minimalism)**：卡片居中，背景可以是模糊的毛玻璃效果或纯色，突出内容。
*   **排版**：大字体，高行高，Markdown 渲染优化。
*   **动效**：
    *   翻页时有轻微的滑入滑出动画。
    *   悬停查看原文时，Tooltip 有淡入淡出效果。
