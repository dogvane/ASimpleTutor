# 待确认问题与后续迭代

## 1. 待确认问题（实现前需要定）

1. 书籍目录扫描是递归还是仅当前目录？是否需要排除某些子目录？
2. 知识点“按章节组织展示”的具体形式：章节树 vs 扁平列表默认？
3. “关联拓展”的关联类型集合：前置依赖/对比/包含 是否足够？
4. 题型优先级：选择题/填空/简答 哪个先做？是否允许只做一种题型？
5. 是否允许使用外部知识（书外补充）？若允许，需要在 UI 明确标注来源。
6. 是否需要缓存落盘？如果需要，JSON 还是 SQLite？缓存放哪里？
7. 参考书目目录的约定：默认目录名是什么（`references`/`参考书目`）？是否允许配置多个？
8. 参考书目目录的排除规则：仅扫描排除，还是连同主检索/RAG 一并排除（建议两者都排除）？
9. 后续“深度探索”是否需要为参考资料建立独立索引（ReferenceRagIndex）？其更新/失效策略如何与主书区分？

## 2. 后续迭代方向（非 MVP）

- 多书/多项目管理与切换
- 学习闭环：进度、错题本、学习报告、章节测验
- 更强检索：向量检索、段落级相关性排序
- 更强评测：Rubric 打分、分层难度、自适应出题
- 协作与分享：导出学习笔记、生成卡片
