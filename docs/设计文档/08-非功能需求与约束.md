# 非功能需求与约束（MVP）

## 1. 性能

- 首次扫描：目标在可接受时间内完成（视书籍规模）；UI 需展示进度/提示
- 学习内容/习题生成：单次生成尽量在数秒内（依赖模型/硬件）

## 2. 可解释性

- 学习与练习都应能回溯到原文片段
- 避免“凭空补充”书中没有的结论

## 3. 可维护性

- 模块边界清晰：扫描/知识构建/内容生成/练习/存储/UI
- 数据结构版本化（schema_version）

## 4. 可观测性（MVP）

- 日志：
  - 扫描到的文件列表
  - Agent 调用耗时
  - 失败原因与降级路径

## 5. 约束与不做

- 单书目录
- 不保存学习进度、成绩与历史
