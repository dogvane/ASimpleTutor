# Research-Learning 模块功能需求文档

## 1. 项目目标

基于书籍文档的智能学习系统（最小 MVP），输入为指定目录中的一组 .md 文档。
系统利用 Agent 对文档进行扫描，建立知识体系，提取知识点，并为选定知识点生成学习内容与练习题，帮助用户高效学习书籍内容。

### MVP 范围说明

- 本阶段默认只面向“单本书/单个目录”的学习场景（不做多项目管理/切换）。
- 本阶段不做学习闭环能力（例如：学习进度、错题本、学习报告、章节回顾与考核）。

## 2. 核心功能需求

### 2.1 知识体系扫描与建立

- 程序启动时自动扫描指定目录中的.md文档
- 基于Agent分析文档内容，建立书籍的知识体系
- 提取知识点间的层次关系和关联性

### 2.2 知识点展示与选择

- 在主界面展示提取出的知识点列表
- 支持用户点击知识点开始学习
- 提供搜索框，用户可输入想了解的具体知识点
- 知识点按章节或重要性进行组织展示

### 2.3 知识点学习方式

用户选择某个知识点后，系统提供以下最小可用的学习方式，帮助用户从“快速理解”到“结合原文掌握”，再到“简单自测”。

- **精要速览**：Agent 生成该知识点的精要总结（定义/要点/常见误区）
- **原文对照**：展示书籍文档中与该知识点相关的原始段落，便于对照理解
- **层次化展开**：同一知识点支持分层查看（概览→详细→深入）
- **关联拓展（可选）**：展示与当前知识点有关联的其他知识点，支持一键跳转
- **快速自测（可选）**：生成 1~3 道与当前知识点相关的小题，用于即时检验理解（不做进度/成绩沉淀）

### 2.4 习题生成与练习

- 在用户进入某个知识点学习时，按需生成与该知识点相关的习题
- 习题类型包括选择题、填空题、简答题等
- 用户可基于理解程度进入习题环节
- 习题内容基于知识点内容自动生成

### 2.5 练习反馈（最小可用）

- 用户提交答案后，给出正确性反馈
- 对于非唯一答案题型，提供参考答案/解题要点（由 Agent 生成）

---

**目标**：创建一个基于书籍文档的智能学习系统（最小 MVP），通过 Agent 技术实现知识点提取、内容生成与练习反馈，提供可用的书籍学习体验。
**agent重点**：知识体系建立、内容总结、习题生成、练习反馈。
